## 1. 앱 개요

**Mystery Factory Pro**는 일본 미스테리/도시전설 유튜브 쇼츠를  
주제 하나만 입력하면 **완전 자동**으로 제작해주는 AI 영상 생성 시스템입니다.

### 핵심 흐름

```
주제 입력
  → ① Gemini AI   — 일본어 스크립트 + 기획안 자동 생성
  → ② Imagen 3    — 2K 고화질 세로형(9:16) 스토리 이미지 최대 15장 생성
  → ③ Gemini TTS  — 일본어 나레이션 음성(WAV) 자동 생성
  → ④ Veo 3.1     — 인트로 훅(Hook) 영상(MP4) 생성
  → ⑤ ZIP 다운로드 → Python render.py 실행 → 최종 영상 mp4 완성
```

### 사용 AI 모델

| 역할 | 모델 |
|---|---|
| 스크립트 생성 | Gemini 3 Pro Preview |
| 이미지 생성 | Gemini 3 Pro Image Preview (2K, 9:16) |
| 음성 합성 | Gemini 2.5 Flash TTS (Puck 보이스) |
| 훅 영상 생성 | Veo 3.1 Fast Generate Preview |

## 2. 환경 설정 및 실행

### 2-1. 필수 조건

| 항목 | 요구사항 |
|---|---|
| Node.js | v18 이상 |
| Python | v3.9 이상 |
| pip 패키지 | moviepy, pillow, numpy |
| API 키 | Google Gemini API Key |
| BGM 파일 | MP3 또는 WAV 형식 |

### 2-2. Python 패키지 설치

```bash
pip install moviepy pillow numpy
```

### 2-3. API 키 설정

프로젝트 루트의 `.env.local` 파일을 열고 아래와 같이 설정합니다:

```
VITE_GEMINI_API_KEY=여기에_Gemini_API_키를_입력
```

> 💡 **API 키 발급:**  
> https://aistudio.google.com 접속 → [Get API Key] 클릭 → 발급된 키 복사

### 2-4. 앱 실행

```bash
# 의존성 설치 (최초 1회)
npm install

# 개발 서버 실행
npm run dev
```

실행 후 브라우저에서 `http://localhost:5173` (또는 표시된 포트) 접속

### ① Content Niche (페르소나 선택)

영상의 **스타일과 톤**을 결정하는 채널 컨셉을 선택합니다.

| 페르소나 | 일본어명 | 특징 |
|---|---|---|
| 🚀 **성공 자극** | 成功への刺激 | 성공/비즈니스 마인드셋 콘텐츠 |
| 👻 **도시전설의 진실** | 都市伝説の真実 | 일본 미스테리/도시전설 (기본 추천) |
| 😂 **폭소 스낵** | 爆笑スナック | 공감형 유머/에피소드 |
| 🌙 **한밤중의 힐링** | 真夜中の癒やし | 감성/힐링 콘텐츠 |

> 💡 미스테리 영상 제작 시 **「都市伝説の真実」** 페르소나를 선택하세요.

---

### ② JP Mystery Presets (일본 괴담 특화)

34개의 엄선된 일본 괴담 중 **8개가 랜덤 표시됩니다.**

- 클릭하면 해당 괴담이 **주제 입력란에 자동 입력**됩니다
- 우측 **「새로고침」 버튼**을 누르면 중복 없이 새로운 8개가 등장합니다
- 34개를 모두 본 후에는 자동으로 풀이 리셋됩니다

**포함된 주요 괴담 목록:**

> 赤いコート, 人面犬, 口裂け女, 鏡の中の他人, 呪いの絵画, 地下鉄の幽霊,  
> 木の上の笑い声, 一本足の女, 電話鬼, 深夜のコンビニ, 死後のSNS ... 외 23개

---

### ③ Master Topic (주제 입력)

자유롭게 주제를 입력합니다. **한글 또는 일본어 모두 가능**합니다.

| 입력 예시 |
|---|
| `새벽 3시 교차로마다 나타나는 붉은 코트의 여자` |
| `심야 12시, 거울 속 내 모습이 0.5초 늦게 움직인다` |
| `呪いの絵画 — 경매 때마다 소유자가 사망하는 그림` |

> 💡 주제는 **구체적이고 자극적일수록** AI가 더 흥미로운 스크립트를 생성합니다.

---

### ④ Viral Hook FX (영상 효과)

훅 영상에 적용되는 **공포 분위기 시각 효과**입니다. 복수 선택 가능합니다.

| 효과 | 설명 |
|---|---|
| 👁 **암시(暗視)** — 나이트비전 | 녹색 야간 투시 필터 + 스캔라인 |
| ➕ **追跡** — 트래킹 | 감시 카메라 코너 마커 오버레이 |
| ⚡ **振動** — 지터 효과 | 필름 흔들림 노이즈 효과 |

---

### ⑤ Horror SFX (공포 음향 효과)

**numpy로 수학적으로 생성된 공포 음향**이 장면 전환 시 자동 삽입됩니다.  
외부 파일 없이 Python이 직접 생성합니다.

| SFX | 효과 |
|---|---|
| 🔴 恐怖ノイズ | 저주파 노이즈 + 럼블 |
| 💜 心臓の鼓動 | 심장 박동음 (1초 간격 더블 펄스) |
| 💨 切替効果 | 장면 전환 휘파람 다운스윕 |
| 🌑 低音振動 | 30~55Hz 저음 공명 |
| ⚡ 静電気 | 정전기 버스트 |

> ⚙️ 선택한 SFX는 **훅 이후, 중간, 엔딩 직전** 3개 지점에 자동 배치됩니다.

---

### ⑥ Hook Title Text (훅 타이틀 문구)

훅 영상 **좌측에 세로로 표시되는 타이틀**을 변경할 수 있습니다.

- 기본값: **「都市伝説」**
- 최대 **6자** 입력 가능
- 진한 빨간색 + 흰색 외곽선으로 표시됩니다

| 입력 예시 |
|---|
| 都市伝説 |
| 呪われ |
| 怪談話 |
| 実話怪 |

---

### ⑦ Background Music — BGM 업로드 (필수)

MP3 또는 WAV 파일을 선택합니다.  
BGM은 나레이션 음량의 약 **18% 볼륨**으로 자동 믹싱됩니다.

> ⚠️ **BGM 파일이 없으면 영상 내보내기가 불가능합니다.**  
> 저작권 없는 BGM은 [Pixabay Music](https://pixabay.com/music/) 등에서 무료 다운로드 가능합니다.

## 4. AI 생성 프로세스

**「AI GENERATE」 버튼**을 누르면 자동으로 아래 단계가 순서대로 실행됩니다.

| 단계 | 내용 | 소요 시간 |
|---|---|---|
| **① 스크립트 생성** | Gemini가 일본어 스크립트 + 15개 이미지 프롬프트 생성 | 30~60초 |
| **② 스토리 이미지 생성** | Imagen 3로 2K 9:16 이미지 최대 15장 생성 | 2~5분 |
| **③ 나레이션 생성** | Gemini TTS (Puck 보이스)로 WAV 음성 생성 | 15~30초 |
| **④ 훅 이미지 생성** | 인트로 훅용 이미지 1장 별도 생성 | 30~60초 |

> 전체 생성 시간: 약 **5~10분**

## 5. 결과물 확인 및 편집

생성이 완료되면 우측 메인 영역에 결과물이 표시됩니다.

### 5-1. 스크립트 카드

- **일본어 스크립트** 전문 확인 가능
- 한국어 번역 스크립트 대조 가능
- 유튜브 제목 및 설명문 확인

### 5-2. 스토리보드 이미지

- 생성된 이미지들을 **그리드 형태**로 미리보기
- 각 이미지 **클릭** → 확대보기 + 프롬프트 수정 가능
- ✏️ **재생성 버튼**: 특정 이미지만 다시 생성
- 📋 프롬프트 자유 편집 후 재생성 가능

### 5-3. 훅 영상 생성 (선택사항)

- Production Center 탭에서 **「Veo 3.1 영상 엔진 가동」** 버튼 클릭
- Veo 3.1이 훅 이미지를 기반으로 **7초 영상(MP4)** 생성
- 소요 시간: **1~3분**

> 💡 훅 영상은 선택사항입니다. 생략 시 훅 이미지(정지화면)가 자동으로 사용됩니다.

### 5-4. 음성 미리듣기

VideoPreview 패널에서 **▶ 재생 버튼** 클릭 → 나레이션 + BGM 믹싱 미리 청취 가능

## 6. Export & 최종 영상 렌더링

### 6-1. Production Kit ZIP 다운로드

Production Center 탭에서 **「ZIP 다운로드」** 버튼 클릭

다운로드된 ZIP 파일에는 아래 파일이 포함됩니다:

| 파일명 | 내용 |
|---|---|
| `hook.mp4` | Veo 3.1 생성 훅 영상 (또는 정지 훅 이미지) |
| `story_0.png ~ story_14.png` | 스토리 이미지 (최대 15장) |
| `narration.wav` | 일본어 AI 나레이션 음성 |
| `subtitles_ja.srt` | 자막 파일 (타이밍 자동 계산) |
| `bgm.mp3` | 업로드한 BGM 파일 |
| `render.py` | 영상 합성 Python 스크립트 |
| `youtube_metadata.txt` | 유튜브 제목/설명/해시태그 |

### 6-2. ZIP 압축 해제 및 폴더 준비

ZIP을 작업 폴더에 압축 해제합니다.  
모든 파일이 **같은 폴더 안에** 있는지 확인합니다.

### 6-3. Python 렌더링 실행

```bash
# Windows 명령 프롬프트 또는 PowerShell
cd [압축 해제한 폴더 경로]
python render.py
```

또는 프로젝트 루트의 **`RENDER_VIDEO.bat`** 파일을 더블클릭합니다.

> ⏱ 렌더링 소요 시간: PC 사양에 따라 **5~20분**

### 6-4. 최종 파일 확인

렌더링 완료 후 동일 폴더에 **`final_output.mp4`** 파일이 생성됩니다.

## 8. 오류 대처 가이드

### ❌ API_KEY_MISSING
**원인:** `.env.local`에 API 키가 없거나 잘못 입력됨  
**해결:** `.env.local` 파일에 `VITE_GEMINI_API_KEY=키값` 형식으로 저장 후 앱 재시작

---

### ❌ QUOTA_EXCEEDED
**원인:** Gemini API 무료 할당량 초과  
**해결:** 잠시(5~10분) 대기 후 재시도. 앱이 자동으로 최대 5회 재시도함

---

### ❌ VIDEO_MODEL_NOT_FOUND
**원인:** Veo 3.1 모델 접근 권한 없음  
**해결:** AI Studio에서 API 키를 재선택하거나, 훅 영상 생성을 건너뛰고 훅 이미지만 사용

---

### ❌ render.py 실행 오류
**원인:** Python 패키지가 설치되지 않음  
**해결:**
```bash
pip install moviepy pillow numpy
```

---

### ❌ 이미지가 일부만 생성됨
**원인:** API 할당량 부족 또는 네트워크 일시 오류  
**해결:** 스토리보드에서 누락된 이미지만 개별 선택 → **✏️ 재생성 버튼** 클릭

---

### ❌ 나레이션이 너무 빠르거나 끊김
**원인:** 스크립트 길이 이슈  
**해결:** 스크립트가 1500자 내외인지 확인. 너무 긴 경우 스크립트 일부 삭제 후 재생성

## 9. 자주 묻는 질문 (FAQ)

**Q. 영상을 만드는 데 비용이 드나요?**  
A. Gemini API 무료 티어 한도 내에서는 무료입니다. 대량 생성 시 유료 플랜이 필요할 수 있습니다.

---

**Q. 하루에 몇 개까지 영상을 만들 수 있나요?**  
A. 무료 티어 기준 약 5~10개 정도 생성 가능합니다. API 할당량은 매일 자정 초기화됩니다.

---

**Q. 쇼츠에 썸네일이 필요한가요?**  
A. 아니요. 유튜브 쇼츠(60초 초과 ~ 3분 미만)는 썸네일이 불필요합니다. 첫 프레임이 자동으로 미리보기로 표시됩니다.

---

**Q. BGM 음량이 너무 크거나 작아요.**  
A. `render.py` 파일 안의 `volumex(0.18)` 값을 조정하세요.  
- 크게: `0.25` 이상  
- 작게: `0.10` 이하

---

**Q. 훅 영상 없이 정지 이미지만 사용해도 되나요?**  
A. 네. Veo 3.1 생성을 건너뛰면 훅 이미지(정지화면)가 자동으로 사용됩니다.

---

**Q. 생성된 이미지 프롬프트를 직접 수정할 수 있나요?**  
A. 네. 각 이미지 카드를 클릭하면 프롬프트 편집창이 열립니다. 수정 후 재생성 버튼을 누르세요.

---

**Q. 완성된 영상을 바로 유튜브에 올릴 수 있나요?**  
A. 네. `final_output.mp4`를 유튜브 쇼츠에 바로 업로드 가능합니다.  
제목·설명·해시태그는 `youtube_metadata.txt` 내용을 그대로 복붙하세요.
